## 1 引言

### 1.1 目的

本文档详细描述"约拍到出片"服务平台的功能需求、非功能需求及需求优先级，为软件开发、测试、验收及后续迭代提供统一依据。本文档面向产品经理、开发工程师、测试工程师、UI/UX设计师及项目管理人员。

本文档中所有需求均标注来源：**[AI生成]** 表示由AI直接生成且经团队确认合理；**[人工补充]** 表示基于问卷或访谈数据由团队补充；**[AI生成+人工修改]** 表示AI生成后经团队基于调研数据修正。

### 1.2 范围

#### 1.2.1 产品名称

约拍到出片（SnapMatch）服务平台

#### 1.2.2 产品功能

整合摄影师、化妆师与拍摄需求方的全链路对接流程，实现**找人—沟通—预约—交易—交付—评价**六大核心环节的标准化与透明化。解决当前大学生约拍市场信息碎片化、价格不透明、信任缺失、交易无保障等核心痛点。

#### 1.2.3 应用范围

目标用户：18–28岁大学生及年轻群体（问卷受访者以大学生为主）。

支持终端：微信小程序（优先），后续迭代iOS/Android原生APP。

覆盖场景：个人写真、旅行跟拍、闺蜜/情侣/家庭合拍、Cosplay约拍、毕业照、日常街拍等（问卷Q4场景分布数据支撑）。

#### 1.2.4 不包含范围

本版本不包含：线下拍摄场地租赁服务、专业商业摄影（产品摄影/婚纱摄影）、视频拍摄与剪辑服务、实体相册制作与邮寄服务。

## 2 总体描述

### 2.1 产品视角

本产品是一个独立的第三方服务对接平台，不直接提供摄影或化妆服务，而是作为中介连接需求方与服务方。当前大学生约拍需求散落在小红书、微信群等渠道（问卷显示小红书/抖音是50%用户的主要渠道），存在信息碎片化、价格不透明、信任缺失等核心问题。本平台通过标准化服务流程、建立双向信用评价体系、提供担保交易保障，填补这一市场空白。

### 2.2 用户特征

#### 2.2.1 需求方

| **维度** | **描述** | **数据来源** |
| --- | --- | --- |
| 身份 | 以大学生为主，占问卷受访者58.54% | 问卷Q1 |
| 预算 | 集中在100–500元区间（66%以上） | 问卷Q12 |
| 行为 | 活跃于小红书/抖音，注重审美风格匹配 | 问卷Q5, 访谈 |
| 核心顾虑 | 成片质量（79.17%）、被骗钱（58.33%）、人身安全（33.33%） | 问卷Q13 |
| 现状 | 46.67%无出片经历，需求转化率低 | 问卷Q2 |

#### 2.2.2 服务方

| **维度** | **描述** | **数据来源** |
| --- | --- | --- |
| 身份 | 以大学生兼职摄影师和独立摄影师为主 | 问卷Q1, 访谈 |
| 接单现状 | 72.73%处于"考虑中"阶段，尚未开始接单 | 问卷Q15 |
| 核心困扰 | 不知道去哪找客户（54.55%）、不确定定价（63.64%） | 问卷Q22 |

| 关键诉求 | 用户信用参考（筛选靠谱客户）、明确责任边界 | 访谈笔记三 |

#### 2.2.3 平台管理员

负责平台日常运营、内容审核、用户投诉处理、纠纷仲裁、系统维护等工作。

### 2.3 约束

1. **技术约束**：优先开发微信小程序，兼容微信8.0及以上版本。
2. **法律约束**：严格遵守《个人信息保护法》《电子商务法》《民法典》肖像权条款。
3. **支付约束**：仅支持微信支付（小程序生态内优先）。
4. **内容约束**：禁止发布色情、暴力、政治敏感等违法违规内容。
5. **时间约束**：V1.0版本开发周期约3个月。

### 2.4 假设和依赖

1. 假设微信小程序和支付接口稳定可用。
2. 假设用户拥有智能手机并能正常使用微信。
3. 假设服务方拥有基本拍摄设备。
4. 依赖第三方云存储服务存储用户上传的照片。
5. 依赖第三方短信服务发送验证码和通知。
