## 用例一：UC-05 下单支付（担保交易）

| 项目 | 内容 |
|------|------|
| **用例编号** | UC-05 |
| **用例名称** | 下单支付（担保交易） |
| **参与角色** | 需求方（主）、服务方（辅） |
| **需求来源** | FR-5.1~5.5 |
| **优先级** | Must |
| **简要描述** | 需求方与服务方通过在线沟通达成一致后，需求方根据服务方发送的报价单进行支付，资金由平台托管，拍摄完成并确认收货后再结算给服务方。 |

### 前置条件

1. 需求方已注册登录。
2. 服务方已完成入驻认证且处于接单状态。
3. 双方已通过站内消息沟通完成需求确认。
4. 服务方已通过聊天窗口向需求方发送了正式的「报价单」（包含拍摄日期、时长、地点、原片张数、精修张数、交付截止日期、总价）。

### 基本流（主成功场景）

1. 需求方在聊天窗口中点击服务方发送的报价单，查看详细条款。
2. 系统展示报价单详情，包含：拍摄日期与时间、拍摄地点、服务内容（原片数量、精修数量）、交付截止日期、照片使用授权范围、总价。
3. 需求方确认报价单内容无误，点击「确认并支付」。
4. 系统生成订单（状态：待支付），跳转至微信支付界面。
5. 需求方完成微信支付。
6. 系统收到支付成功回调，将订单状态更新为「已支付-待拍摄」，资金进入平台托管账户。
7. 系统向双方发送订单确认通知（站内消息 + 微信服务通知）。
8. 服务方的档期日历自动标记该时段为「已预约」。

### 替代流

- **3a. 需求方对报价单有异议：**
  - 3a.1 需求方点击「协商修改」，在聊天中与服务方沟通调整。
  - 3a.2 服务方修改后重新发送报价单，回到步骤1。

- **5a. 支付失败（余额不足/网络异常）：**
  - 5a.1 系统提示支付失败原因，订单保持「待支付」状态。
  - 5a.2 需求方可在24小时内重新发起支付。
  - 5a.3 若24小时内未完成支付，订单自动取消，系统通知双方。

- **6a. 支付回调超时：**
  - 6a.1 系统每隔5分钟主动向支付平台查询订单状态。
  - 6a.2 确认支付成功后，继续执行步骤6~8。
  - 6a.3 若30分钟内仍未确认，提示需求方联系客服。

### 后置条件

1. 订单状态变为「已支付-待拍摄」。
2. 资金由平台托管，需求方和服务方均不可直接操作资金。
3. 双方均可在「我的订单」中查看订单详情。
4. 服务方的档期已更新。
5. 订单关联的报价单条款（包括照片使用授权范围）作为后续纠纷仲裁的依据留存。

## 用例二：UC-08 发起成片质量申诉

| 项目 | 内容 |
|------|------|
| **用例编号** | UC-08 |
| **用例名称** | 发起成片质量申诉 |
| **参与角色** | 需求方（主）、服务方（辅）、平台管理员（辅） |
| **需求来源** | FR-5.7, FR-8.2~8.3 |
| **优先级** | Must |
| **简要描述** | 需求方收到服务方交付的作品后，如认为成片质量与约定不符或与作品集展示水平严重不符，可在确认收货前发起质量申诉，由平台介入仲裁。此用例是调研中发现的核心痛点：问卷显示成片质量不满意是最大卡点（50%），79.17%用户担心成片效果不符预期。 |

### 前置条件

1. 订单状态为「已交付-待确认」（服务方已上传作品）。
2. 需求方已在线查看了服务方交付的作品。
3. 需求方尚未点击「确认收货」（即7天自动确认期内）。

### 基本流（主成功场景）

1. 需求方在订单详情页点击「成片不满意，申请仲裁」。
2. 系统展示申诉表单，包含以下必填项：
   - 不满意的类型（多选）：修图风格不符、构图/光影质量差、精修数量不足、交付格式不对、其他。
   - 问题描述（文字，至少20字）。
   - 上传对比证据（如：报价单中的参考图 vs 实际成片，服务方作品集展示 vs 实际交付）。
3. 需求方填写申诉表单并提交。
4. 系统将订单状态更新为「申诉处理中」，冻结7天自动确认倒计时。
5. 系统通知服务方：「需求方对本单成片质量提出申诉，请在48小时内回复。」
6. 服务方在48小时内查看申诉内容并提交回复（可附加说明和补充证据）。
7. 平台管理员收到仲裁任务，审核双方提交的证据。
8. 管理员在48小时内作出仲裁决定，可能结果包括：
   - **全额退款**：成片质量与约定严重不符（如原片数量不足、精修明显敷衍）。
   - **部分退款**：存在一定质量问题但不构成严重不符。
   - **驳回申诉**：成片质量符合约定标准，需求方主观偏好不构成退款理由。
   - **要求返修**：服务方在约定时间内重新修图交付。
9. 系统将仲裁结果通知双方，并执行相应的资金操作。

### 替代流

- **6a. 服务方48小时内未回复：**
  - 6a.1 系统自动通知管理员，标注「服务方未回应」。
  - 6a.2 管理员仅基于需求方提交的证据进行仲裁，倾向于保护需求方权益。

- **8a. 仲裁结果为「要求返修」：**
  - 8a.1 服务方需在新约定的截止日期前重新上传修改后的作品。
  - 8a.2 需求方收到返修作品后，可选择「确认收货」或「再次申诉」。
  - 8a.3 若再次申诉，管理员直接介入处理，不再等待服务方回复。

- **8b. 任一方对仲裁结果不满：**
  - 8b.1 可在收到仲裁结果后24小时内提交一次申诉复核请求。
  - 8b.2 由另一位管理员（非原仲裁人员）进行复核。
  - 8b.3 复核结果为最终裁决。

- **3a. 需求方在提交前取消申诉：**
  - 3a.1 订单恢复「已交付-待确认」状态，7天倒计时继续。

### 后置条件

1. 仲裁结果被记录到订单历史中，作为信用评分的计算依据。
2. 若判定退款：平台将相应金额退还需求方，服务方信用评分扣减。
3. 若驳回申诉：资金正常结算给服务方，需求方的申诉记录保留但不影响信用。
4. 仲裁过程中的所有证据和沟通记录永久保存。
5. 服务方的仲裁统计（被申诉次数/判定退款次数）影响其主页展示的信用等级。

## 用例三：UC-03 发布约拍需求

| 项目 | 内容 |
|------|------|
| **用例编号** | UC-03 |
| **用例名称** | 发布约拍需求 |
| **参与角色** | 需求方（主）、服务方（辅） |
| **需求来源** | FR-3.1~3.4 |
| **优先级** | Must |
| **简要描述** | 需求方将拍摄需求（场景、风格、时间、地点、预算）发布到平台的「需求大厅」，由感兴趣的服务方主动响应。此功能解决了调研中发现的核心障碍：62.5%的未约拍用户不知道去哪找靠谱摄影师，本用例将「需求方主动找人」反转为「服务方主动找客户」，降低需求方的行动门槛。 |

### 前置条件

1. 需求方已注册登录。
2. 需求方当前没有超过3条「进行中」的未成交需求（防刷限制）。

### 基本流（主成功场景）

1. 需求方在首页点击「发布约拍需求」。
2. 系统判断该用户是否首次发布：
   - 若是首次，展示简短的流程引导（3步：填写需求 → 等待响应 → 选择服务方）。
   - 若非首次，直接进入发布表单。
3. 系统展示需求发布表单，包含以下字段：
   - **拍摄场景**（单选）：个人写真、旅行跟拍、闺蜜/情侣合拍、Cosplay、毕业照、其他。
   - **期望风格**（多选标签）：自然抓拍、氛围感、日系清新、复古胶片、韩系精致、其他。
   - **期望日期与时间段**（日期选择器 + 时段）。
   - **拍摄地点**（文字输入 + 地图定位可选）。
   - **预算范围**（下拉选择）：100元以下 / 100~200元 / 200~500元 / 500~1000元 / 1000元以上 / 面议。
   - **参考图片**（可选，最多上传6张）。
   - **补充说明**（可选，自由文字）。
4. 需求方也可以点击「使用拍摄企划模板」，系统提供预填的结构化模板（包含妆造要求、服装风格、姿态偏好等），需求方只需勾选和补充。
5. 需求方填写完成后，点击「发布需求」。
6. 系统对内容进行自动审核（违禁词检测、图片合规检查）。
7. 审核通过后，需求发布成功，状态为「等待响应」。
8. 系统根据需求的场景、风格、地点，向匹配的服务方推送通知：「有一条新的约拍需求与您的服务匹配」。
9. 服务方在「需求大厅」中查看该需求，感兴趣的服务方点击「我想接单」，向需求方发起沟通。
10. 需求方在消息列表中查看响应的服务方，浏览其作品集和信用评分后，选择其中一位开始沟通。

### 替代流

- **6a. 内容审核未通过：**
  - 6a.1 系统提示违规原因（如包含敏感词、图片不合规等）。
  - 6a.2 需求方修改后重新提交，回到步骤5。

- **8a. 72小时内无服务方响应：**
  - 8a.1 系统通知需求方：「暂时没有服务方响应您的需求，建议调整预算或扩大风格范围。」
  - 8a.2 需求方可选择修改需求重新发布，或主动搜索服务方发起沟通。

- **9a. 响应人数过多（超过10人）：**
  - 9a.1 系统按服务方信用评分和风格匹配度排序展示。
  - 9a.2 需求方从列表中选择感兴趣的服务方查看详情。

- **4a. 需求方不使用模板，直接填写自由表单：**
  - 4a.1 跳过模板步骤，继续步骤5。

- **10a. 需求方未找到满意的服务方：**
  - 10a.1 需求方可点击「关闭需求」，需求从大厅下架。
  - 10a.2 也可保持需求发布状态，继续等待新的服务方响应。

### 后置条件

1. 需求成功发布到平台需求大厅，对所有服务方可见。
2. 匹配的服务方收到推送通知。
3. 需求方可在「我的需求」中管理已发布的需求（编辑、关闭、查看响应列表）。
4. 需求有效期为7天，过期自动下架（需求方可续期）。
5. 需求发布记录保留在用户的历史数据中。
