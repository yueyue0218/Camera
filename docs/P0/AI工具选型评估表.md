gi# AI 工具选型评估表

## 团队名称：Action

## 项目名称：Camera 约拍服务平台

## 日期：2026 年 4 月 2 日

---

## 一、评估背景与原则

本项目技术栈为 Vue 3 + Spring Boot + MySQL，团队 4 人，开发周期跨 P0-P7 共 8 个阶段。为规范 AI 工具使用、提升协作效率，现对四类 AI 工具进行评估选型。

### 评估维度

1. **功能匹配度**：是否适配本项目技术栈与开发场景
2. **易用性**：上手难度、IDE 集成度、中文支持
3. **成本**：学生是否可免费或低价使用
4. **数据安全**：是否存在代码泄露风险
5. **社区生态**：更新活跃度、文档完善度

---

## 二、编程助手（代码补全/生成）

### 候选工具评估

| 维度       | GitHub Copilot           | Codex                       | Cursor                      |
| ---------- | ------------------------ | --------------------------- | --------------------------- |
| 功能匹配度 | 支持 Vue/Java，行级/函数级补全 | 支持 Vue/Java，擅长多文件改写与代理式任务 | AI 原生 IDE，整文件级改写能力强 |
| 易用性     | VS Code/IntelliJ 插件成熟  | 提供 CLI + VS Code 扩展 + 云端代理 | 需要切换到专用 IDE            |
| 成本       | 学生包免费                 | 需 ChatGPT Plus 订阅或 API 付费 | Free 版有限额，Pro 约 $20/月  |
| 数据安全   | 代码会上传云端，但企业级可控 | OpenAI 云端处理，可关闭训练数据收集 | 代码上传云端处理             |
| 社区生态   | 生态最成熟，文档多         | 官方生态完整，迭代速度快       | 快速发展中，更新频繁         |

### ✅ 选择：GitHub Copilot（主）+ Codex（补充）

**理由**：
- **GitHub Copilot**：通过学生认证可免费使用，与团队使用的 VS Code / IntelliJ 无缝集成，行内补全响应快，适合日常高频编码场景
- **Codex** 作为补充：擅长跨文件的代理式任务（如"把所有 Controller 改造成统一异常处理"这类多文件重构），适合 P4/P5 阶段集中攻坚。由至少一名已有 ChatGPT 订阅的成员使用，避免全员付费

### ❌ 未选择：Cursor

**理由**：需要全员切换到新 IDE，学习成本高；付费版超出学生预算；现有 VS Code/IntelliJ 已能通过插件满足需求

## 三、对话工具（方案讨论/文档生成/答疑）

### 候选工具评估

| 维度       | Claude                  | ChatGPT                 | DeepSeek                |
| ---------- | ----------------------- | ----------------------- | ----------------------- |
| 功能匹配度 | 长文本理解强，代码能力优秀  | 通用能力强，生态插件多    | 代码与推理能力强        |
| 易用性     | Web 界面简洁             | Web 界面友好，中文流畅    | 中文原生，界面简洁       |
| 成本       | 免费额度有限，Pro $20/月  | 免费额度有限，Plus $20/月 | 完全免费                |
| 数据安全   | 默认不用于训练            | 可关闭训练数据收集        | 需注意国内合规          |
| 适用场景   | 长文档分析、架构设计讨论   | 日常答疑、代码片段解释    | 代码调试、算法题、文档生成 |

### ✅ 选择：DeepSeek（主）+ Claude（辅）

**理由**：
- **DeepSeek** 完全免费，中文原生，满足日常答疑、文档初稿、调试等高频场景，无成本压力
- **Claude** 用于需要长上下文的任务（如阅读长需求文档、整体架构讨论），利用其免费额度即可

### ❌ 未选择：ChatGPT（作为主力）

**理由**：免费版限额较紧，付费版成本高；DeepSeek 已能覆盖其大部分使用场景；国内访问需额外配置

## 四、代码审查辅助工具

### 候选工具评估

| 维度       | CodeRabbit                    | GitHub Copilot Code Review | Codex（代理式审查）        |
| ---------- | ----------------------------- | -------------------------- | -------------------------- |
| 功能匹配度 | 自动 PR 评审，给出改进建议      | 集成在 GitHub，评审 PR 差异  | 可在 PR 上生成审查意见与补丁 |
| 易用性     | GitHub App 安装即用            | Copilot 订阅自带            | 需配置云端代理或 CLI        |
| 成本       | 开源项目免费，私有仓库有限额     | 包含在 Copilot 订阅中        | 需 ChatGPT 订阅或 API 付费  |
| 数据安全   | 代码上传第三方                 | GitHub 生态内                | OpenAI 云端                |
| 契合度     | 学生项目够用                   | 与 Copilot 组合效果好        | 适合复杂变更的深度审查      |

### ✅ 选择：CodeRabbit（主）+ 人工 Review

**理由**：
- 本项目仓库为公开学生项目，CodeRabbit 对公开仓库免费
- 自动分析 PR 变更并给出建议，与 GitHub PR 流程天然集成
- 但 AI Review **不替代** 人工 Review，按《AI 协作契约》仍需至少一名成员人工审核

### ❌ 未选择：仅依赖人工 Review

**理由**：团队全员都是学生，经验有限，容易遗漏常见代码坏味道和安全问题；引入 AI Review 作为第一道过滤可提升效率

## 五、测试辅助工具

### 候选工具评估

| 维度       | GitHub Copilot（测试生成）    | Qodo（原 CodiumAI）           | Codex（测试生成）          |
| ---------- | ----------------------------- | ----------------------------- | -------------------------- |
| 功能匹配度 | 通过 Prompt 生成 JUnit/Jest 用例 | 专注于单元测试，自动生成 + 分析边界 | 可代理式批量生成测试用例    |
| 易用性     | 已有 Copilot 即可使用            | VS Code 插件，一键生成          | 需写 Prompt 或 CLI 命令    |
| 成本       | 随 Copilot 订阅                  | 个人版免费                     | 需 ChatGPT 订阅或 API 付费 |
| 场景覆盖   | 单元测试、简单集成测试           | 单元测试为主，边界分析强        | 单元 + 集成测试均可         |
| 局限       | 需要精心写 Prompt               | 对复杂业务逻辑生成质量一般      | 成本高，不适合全员日常使用  |

### ✅ 选择：Qodo（主）+ Codex（补充）

**理由**：
- **Qodo** 个人版免费，专注测试生成，能自动识别边界情况和异常路径，比通用 Copilot 生成的测试更系统
- 后端 Spring Boot 使用 JUnit 5，前端 Vue 3 使用 Vitest，Qodo 均支持
- **Codex** 作为补充：在 P6 测试阶段集中补齐测试覆盖率时，用代理式批量生成测试更高效

### ❌ 未选择：仅使用 ChatGPT/Claude 手动生成测试

**理由**：对话工具生成测试需手动复制粘贴、缺乏 IDE 集成，效率低；专用测试工具对测试覆盖率和边界场景分析更专业

## 六、最终工具链总结

| 类别      | 主力工具           | 备选/补充工具       | 主要使用阶段            |
| --------- | ----------------- | ------------------ | ----------------------- |
| 编程助手  | GitHub Copilot    | Codex              | P4、P5 编码阶段          |
| 对话工具  | DeepSeek          | Claude             | P0-P7 全阶段            |
| 代码审查  | CodeRabbit + 人工 | Codex 代理式审查   | P4、P5 编码阶段          |
| 测试辅助  | Qodo              | Codex              | P5 编码二 + P6 测试阶段  |

> **成本说明**：Codex 需 ChatGPT 订阅或 API 付费，团队约定由已拥有订阅的成员承担相关任务，避免全员重复付费。

---

## 七、使用规范补充

1. 所有工具使用须遵守《AI 协作契约》中的场景约定
2. 禁止将用户隐私数据、线上密钥、真实业务数据粘贴给任何 AI 工具
3. AI 生成内容须在 Commit message 中以 `[AI-assisted]` 标注
4. 每阶段结束时在 AI 协作反思日志中评估工具实际效果，若不符预期则在下阶段调整选型
5. 发现新工具时可提交团队讨论，经半数以上同意后加入工具链
